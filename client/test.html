Status: Bot runner is active and ready to accept WebRTC connections.
Architecture: Direct in-process bot (no subprocess). Completely free, open-source (Pipecat + SmallWebRTC).

🚀 How to Connect

1

Test Locally (Development)

Use Pipecat Playground or any WebRTC client to connect to this server:

ws://localhost:7860
2

Production Deployment (Fly.io)

Deploy with the included docker configuration:

fly deploy --dockerfile Dockerfile.flyio

Then connect from your React UI or web client to:

wss://your-app.fly.dev
3

Integrate with Your UI

Use any WebRTC library (Daily.co SDK alternatives, PeerJS, or native RTCPeerConnection) to connect. The bot manages audio I/O via SmallWebRTC transport.

No extra fees or API costs - everything is self-hosted and open-source.

🔧 Quick Reference

Bot endpoints:
• GET / - Landing page
• GET /health - Health check
• GET /status - Active sessions
• POST /start_bot - Create session
• POST /stop_bot/{session_id} - Close session

WebRTC connection:
• Client connects to ws://localhost:7860 (Pipecat handles SmallWebRTC protocol)
• Bot processes audio in real-time
• Tools (booking functions) available during conversation
